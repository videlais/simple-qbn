Plot Fragments and Author Goals (1985)

Term

Michael Lebowitz (1984, 1985) created Universe, a story-generation system designed to produce ongoing, soap-opera-style narratives. Where earlier work simulated characters from the inside, Universe selected from a library of authored plot fragments whose use depended on whether their conditions matched the current cast of characters and the system’s author goals.

A plot fragment in Universe carried preconditions, a set of characters it required, and constraints those characters had to satisfy. Lebowitz (1985) described how fragments were chosen to advance author-level goals rather than character goals, fitting available fragments to the current state of the world:

Plot fragments include the characters needed, constraints on those characters, subgoals to be achieved, and the events that make up the fragment. A fragment can be used only when its constraints are satisfied by the current cast and situation.

- Lebowitz (1985)

This produced several concepts central to later quality-based narratives:

  • Plot fragment: an authored chunk of story, comparable to a storylet.
  • Constraints: requirements on character attributes and world values before a fragment can be used.
  • Author goals: higher-level objectives that guide which available fragment is chosen.
  • Character attributes: the values mutated by fragments and checked by later constraints.

By gating authored fragments behind constraints over a stored cast of values, Universe (1985) is one of the closest pre-1990s ancestors of the quality-based and storylet structures, differing mainly in that selection was driven by author goals instead of player choice.
